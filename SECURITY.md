## 报告安全漏洞

我们非常重视安全问题。如果您发现了安全漏洞，请按照以下流程报告：

### 负责任的披露

1. **不要**在公开的 GitHub Issues 中报告安全漏洞
2. 通过以下方式私下报告：
   - 发送邮件至：lingssc@outlook.com
   - 或使用 GitHub 安全公告功能（如果已启用）

### 报告内容

请在报告中包含以下信息：

- **漏洞描述**：清晰描述漏洞的性质
- **影响范围**：哪些版本受到影响
- **复现步骤**：详细的复现说明
- **可能的解决方案**（可选）：如果您有修复建议
- **您的联系方式**：以便我们与您沟通

### 响应流程

1. **确认收到**：我们会在 48 小时内确认收到您的报告
2. **调查评估**：我们会评估漏洞的严重性和影响范围
3. **修复开发**：开发修复方案
4. **安全更新发布**：发布安全更新
5. **公开披露**：在修复后公开披露漏洞信息（会注明您的贡献，如果您愿意）

### 漏洞处理时间线

| 严重程度 | 目标响应时间 | 目标修复时间 |
|----------|--------------|--------------|
| 严重 | 24 小时内 | 7 天内 |
| 高 | 48 小时内 | 14 天内 |
| 中 | 72 小时内 | 30 天内 |
| 低 | 1 周内 | 下次发布时 |

## 安全最佳实践

### 部署安全

1. **修改默认密码**
   - 首次部署时必须修改默认管理员密码
   - 使用强密码（至少 12 位，包含大小写字母、数字和特殊字符）

2. **保护配置文件**
   - 设置配置文件权限为 600：`chmod 600 frp-console.conf`
   - 不要将实际配置文件提交到版本控制
   - 定期更换 SECRET_KEY

3. **网络安全**
   - 使用 HTTPS 部署
   - 配置防火墙规则，仅开放必要的端口
   - 考虑使用反向代理（如 Nginx）

4. **定期更新**
   - 及时更新到最新版本
   - 关注安全公告

### 数据安全

- 定期备份数据库
- 安全存储备份文件
- 考虑启用数据库加密

## 安全功能

FRP Console 已实现以下安全功能：

- ✅ 密码使用 PBKDF2-HMAC-SHA256 哈希（100,000 次迭代）
- ✅ 随机盐生成
- ✅ 恒定时间密码比较（防止时序攻击）
- ✅ 登录速率限制
- ✅ CSRF 保护
- ✅ Session 安全设置（HttpOnly, SameSite）
- ✅ 密码复杂度验证
